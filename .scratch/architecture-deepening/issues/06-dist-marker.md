# Mark `dist/<version>/` outputs as generated

## Context

`scripts/build-plugin-artifacts.mjs` produces `plugins/<plugin>/dist/<version>/` directories containing copies of `release.json`, `claude/plugin/`, `codex/plugin/`, and `codex/marketplace.json`. These are derived from the Per-plugin Manifests and Marketplace Registries; they are **not** authoritative.

CLAUDE.md says "do not hand-edit." There is no in-tree marker. A maintainer who opens `dist/<v>/codex/marketplace.json` looking for the canonical Codex registry would not realise it's a stale projection.

## Change

In every `dist/<version>/` directory, write a `GENERATED.md` marker file:

```md
# Generated artifacts

This directory is regenerated by `scripts/build-plugin-artifacts.mjs` (or `npm run build:artifacts` from this plugin's directory).

**Do not hand-edit any file in this directory.** Edits are clobbered on the next build.

Source of truth:
- Per-plugin Manifests: `../../.claude-plugin/plugin.json`, `../../.codex-plugin/plugin.json`
- Skills: `../../skills/`
- Marketplace Registries: repo-root `.claude-plugin/marketplace.json` and `.agents/plugins/marketplace.json`
```

If #01 lands first, fold `dist/` projection into `MarketplaceModel.write_dist(version)` so the build script becomes a thin wrapper.

## Rationale

Weak deletion test — this is hygiene, not a deepened module. Worth a marker file because the cost is one file per dist directory and the failure mode (silent shadow truth) is real.

## Sequencing

Independent — can ship before, after, or alongside #01.
